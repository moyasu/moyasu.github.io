甄选镜像过滤方案调研

问题背景

客户自定义一个部署好某个具体应用的镜像，然后要共享给其他客户使用。供其他客户能看到这个镜像并且可以使用它来建虚机。目前 OpenStack Glance 服务中的私有镜像别的租户默认是看不到的。

OpenStack Glance 现状

Glance 的镜像有一个 visibility 属性字段，允许的值包括：public、private、community 和 shared。默认的值为 shared。

  • public

    公共镜像，默认情况下由平台管理员创建可供所有用户可见和使用。

  • private

    自有镜像，只对镜像所有者可见。

  • community

    镜像可以供所有用户使用。

  • shared

    镜像所有者可以完全访问和使用它，并且镜像的成员也可以使用它。

拟定方案

应用厂商自定义镜像（默认 visibility 字段为 shared）后，如果需要上线到甄选商城供其他客户使用，则甄选页面端只需要调用更新接口将这个镜像的 visibility 字段由 shared 更新为 community 即可，这样其他客户（租户）就可以有权限查看到这个镜像详情并且有权限使用它来开通虚机。但是 list-images 里面是看不到的。所以，甄选商城端需要记录这些甄选镜像的 uuid 信息，后续用这个镜像开通应用都需要传送镜像的 uuid。

因为默认其他租户（客户）list-image 列表中看不到这些甄选镜像，所以天翼云 console 端就不需要在公共镜像列表端进行过滤屏蔽掉甄选镜像了。

[root@allinone ~]# cat update_image.py 
import openstack

conn = openstack.connect(cloud="allinone")

image1 = conn.image.find_image("424aae1f-ff7f-43dd-8bf9-4807dadfe5ba")

print("********before********")
print(image1.to_dict()['visibility'])

conn.image.update_image("424aae1f-ff7f-43dd-8bf9-4807dadfe5ba", **{"visibility": "community"})

image1 = conn.image.find_image("424aae1f-ff7f-43dd-8bf9-4807dadfe5ba")

print("********after********")
print(image1.to_dict()['visibility'])

[root@allinone ~]# python update_image.py 
********before********
shared
********after********
community

创建虚机

import openstack


conn = openstack.connect(cloud='allinone')

server = conn.compute.create_server(
    name="demo2_vm",
    image_id="bf0a16fa-9519-4319-a79c-801862672e66",
    flavor_id="2b8a00c2-22e8-4628-b7b5-ec7d46525d1f",
    network=[{"uuid": "411abc23-2e33-4f4b-afd7-bd10c530ec43"}],
)

for server in conn.compute.servers():
    print(server)
